An agent auditing Seneschal found its governance manifest contradicting itself,
and its scanner crying wolf on localhost. Both are fixed, and both now have a
test that makes the mistake impossible to reintroduce silently.

```bash
pip install --upgrade seneschal
```

---

## The manifest contradicted itself

`TOOL_MANIFEST.json` listed **17 of 22** modules as both `planned` and
`implemented`. It is a governance artifact — the document that says what the tool
is — and its own health gate checked only that the JSON *parsed*, never that it
made *sense*. So the exact drift Seneschal exists to prevent sat inside
Seneschal's own manifest, because nothing read the document.

The 17 are removed from planned; five genuinely-unbuilt entries remain. Two tests
now enforce it: planned and implemented must be disjoint, and a planned module
must not already exist as a file.

## Loopback is not exfiltration

`scan` flagged any `http(s)://` as "external link present", so a reference to
`http://127.0.0.1` — where a local-first server runs — tripped the finding. For an
exfiltration-aware scanner, a false positive on localhost trains people to ignore
it. Loopback and the RFC1918 private ranges are now excluded; only genuinely
off-machine hosts are flagged.

## Also

- Default state and ledger paths renamed `.robinhood/` → `.seneschal/`. The
  rename reached the code months ago and never reached the config.
- A CLI test that depended on whether Ollama happened to be running is now
  deterministic.

## Verified, not assumed

The audit also reported that `trust` was always `low`. It was checked and is not
a bug: `trust` derives from `source`, and defaults to low because `scan` defaults
`--source` to external — the correct safe default for a security tool.
Auto-trusting local input would have been the regression. It stays.
