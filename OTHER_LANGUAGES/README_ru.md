# Seneschal

[![PyPI](https://img.shields.io/pypi/v/seneschal?label=pypi&color=blueviolet)](https://pypi.org/project/seneschal/)
[![License](https://img.shields.io/badge/license-MIT-green)](../LICENSE)

Seneschal — это локальный слой контроля, который делает работу с ИИ дешевле и безопаснее.

Он решает, какой контекст отправлять, что оставить локально, что заблокировать и заслуживает ли задача более мощной модели. Цель — реальная экономия токенов: меньше вставок всего репозитория, меньше повторных попыток, меньшие промпты, безопаснее область действия инструментов и понятнее передача задач.

> Сенешаль управлял ресурсами дома и санкционировал действия от имени сеньора. Этот инструмент делает ровно то же самое: **распределяет расходы** и **санкционирует полномочия**.

## Возможности

- Оценка бюджета токенов без привязки к конкретному провайдеру
- Упаковка контекста репозитория в рамках явного бюджета
- Рекомендация самого дешёвого пути к модели, достаточного для задачи
- Снимки контекста, чтобы не пересылать неизменённые файлы
- Отбор наиболее релевантного соседнего контекста по лексической мере BM25
- Проверка недоверенного текста и файлов на инъекции промптов и секреты
- Верификация полномочий минимальной привилегии, **подписанных Ed25519**

## Установка

```bash
pip install seneschal
seneschal --help
```

Дополнительные расширения:

```bash
pip install "seneschal[measure]"    # измеренный токенизатор (tiktoken)
pip install "seneschal[security]"   # подписанные полномочия (cryptography)
```

## Подписанные полномочия

Неподписанное разрешение — это просто JSON-файл, который может отредактировать любой процесс, включая самого агента, которого оно должно ограничивать. С подписью Ed25519 брокер отказывает по принципу fail-closed, если подпись отсутствует, сделана недоверенным ключом, файл изменён после подписания (повышение привилегий) или срок действия истёк.

```bash
seneschal keygen
seneschal grant --sign --task-id RH-001 --capability read --allowed-path src/ --out grant.json
seneschal grant --grant-file grant.json --require-signed --task-id RH-001 --action read --path src/main.py
```

## Чем это не является

Это не песочница и не движок выполнения агентов. Он решает, **вызывать** ли модель и **какой** контекст ей отправить; он не ограничивает то, что модель или агент делает дальше с выданным полномочием.
